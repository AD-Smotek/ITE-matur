# Úvod do backendu

## 1) Co je backend

Backend je **serverová část webové aplikace**, která není viditelná pro uživatele.

Zajišťuje:
- zpracování dat
- komunikaci s databází
- přihlašování uživatelů
- logiku aplikace
- bezpečnost

Frontend = to, co vidí uživatel (HTML, CSS, JavaScript)  
Backend = to, co běží na serveru

---

## 2) Jak funguje webová aplikace

1. Uživatel odešle požadavek (např. přihlášení).
2. Server přijme požadavek.
3. Backend zpracuje data.
4. Komunikuje s databází.
5. Vrátí odpověď (HTML, JSON).

Tento proces funguje na principu **klient–server**.

---

## 3) Server a klient

Klient:
- webový prohlížeč (Chrome, Firefox)
- posílá HTTP požadavky

Server:
- přijímá požadavky
- vrací odpovědi
- běží na vzdáleném počítači

Komunikace probíhá pomocí protokolu HTTP nebo HTTPS.

---

## 4) Backendové technologie

Nejčastější jazyky:

- PHP
- JavaScript (Node.js)
- Python
- Java
- C#

Backend běží na webovém serveru (např. Apache, Nginx).

---

## 5) Práce s databází

Backend:
- přijme data z formuláře
- ověří je
- uloží do databáze
- načte data z databáze
- vrátí výsledek uživateli

Například:
- přihlášení uživatele
- registrace
- objednávka v e-shopu

---

## 6) API

API (Application Programming Interface) umožňuje komunikaci mezi aplikacemi.

Backend často vrací data ve formátu JSON:

```json
{
  "jmeno": "Jan",
  "vek": 18
}
```

Používá se například pro:
- mobilní aplikace
- komunikaci frontend ↔ backend
- externí služby

---

## 7) Bezpečnost backendu

Důležité zásady:

- Ověřování dat
- Hashování hesel
- Ochrana proti SQL injection
- Ochrana proti XSS
- Používání HTTPS
- Autentizace a autorizace

Backend je klíčový pro bezpečnost celé aplikace.

---

## 8) Autentizace a autorizace

Autentizace – ověření identity (přihlášení).  
Autorizace – určení oprávnění (co může uživatel dělat).

Například:
- běžný uživatel
- administrátor

---

## 9) Shrnutí

Backend je serverová část aplikace.  
Zpracovává data, komunikuje s databází a zajišťuje bezpečnost.  
Funguje na principu klient–server a používá HTTP komunikaci.  

Bez backendu by webová aplikace nebyla funkční.
